# TUI 双栏浏览与选择设计

从终端用户视角，目标是进入工具后立刻看到一个全屏双栏界面：左边像文件管理器一样浏览当前目录，右边实时显示已选清单，我只需用方向键与空格完成勾选，然后按 `N` 进入下一步配置，无需切换焦点或输入路径。

## 范围

- TUI 改为全屏双栏布局（左：当前目录浏览；右：已选清单）。
- 支持目录与媒体文件选择（视频/图片/GIF）。
- 支持当前目录层级的快速过滤（直接输入触发）。
- 进入下一步前弹出“展开确认”层。
- 目录递归展开，输出规则按“一级目录平铺/单文件平铺到输出目录”。
- 下一步配置默认输出目录 `./output`，可修改。
- 使用 Lip Gloss 统一布局与样式。
- TUI 文案统一使用英文。

## 非目标

- 不实现文件预览（仅显示名称与类型）。
- 不实现高级搜索/模糊匹配（保留扩展接口）。
- 不变更后续编码策略逻辑。

## 交互布局

- 全屏两栏，占满终端窗口。
- 左栏：当前目录列表；仅显示目录与支持的媒体文件。
- 右栏：已选清单（保持选择顺序）。
- 顶部显示“当前目录”和“搜索过滤”行，搜索为直接输入。
- 底部状态区：焦点、已选数量、过滤信息、快捷键提示。

```
┌ 左: 当前目录 ───────────────┬ 右: 已选清单 ───────────────┐
│ 当前目录: ./                │ 已选清单 (2)               │
│ 搜索: cat_                  │                             │
├─────────────────────────────┼─────────────────────────────┤
│ [✓] cat01.gif               │ 1) ./cats/cat01.gif        │
│ [ ] dogs/                   │ 2) ./dogs/dog02.png        │
│ ...                         │ Enter: 下一步  ⌫ 删除       │
└ Space 选中  Enter 进目录  Tab 切焦点 ─────────────────────┘
```

## 键位与行为

- `↑/↓`：移动光标。
- `Enter`：
  - 左栏：进入目录。
  - 右栏：进入下一步配置。
- `Space`：左栏切换选中/取消；左栏显示 `[✓]`。
- `Tab`：切换左右栏焦点（仅为查看/删除/进入下一步）。
- 直接输入字符：更新搜索过滤。
- `Backspace`：
  - 左栏且过滤非空：删除过滤字符。
  - 右栏：删除当前高亮项。
- `Esc`：关闭确认层。
- `q/ctrl+c`：退出。

## 选择模型

- 维护一个 `SelectedSet`（用于判定是否勾选）与一个 `SelectedList`（用于右栏显示与有序处理）。
- 维护一个 `FilterText`（用于左栏过滤显示）。
- 左栏渲染时根据 `SelectedSet` 标记 `[✓]`。
- 右栏删除时同步更新 `SelectedSet`。

## 目录展开与输出规则

- 当选中项包含目录时，在进入下一步前进行递归扫描。
- 弹出确认层展示：目录数量 / 已选文件数 / 展开后文件总数 / 输出目录列表。
- 展开规则：目录内文件递归查找，但输出平铺到该“一级目录”。
- 单独选中的文件：输出平铺到全局输出目录（默认 `./output`），不保留任何目录结构。
- 进入下一步后生成最终任务列表（去重）。

## 状态机与数据流

- `browse`：双栏浏览与选择。
- `confirm`：目录展开确认层。
- `config`：选择处理方式与其它配置（后续界面）。
- `processing`：执行处理。
- `summary`：结果汇总。

数据流：
`browse` 选中 -> `confirm` 扫描 -> `config` 生成任务清单 -> `processing` -> `summary`

## 组件与实现方式

- 左栏：`bubbles/list` 用于目录与文件列表。
- 右栏：`bubbles/list` 用于已选清单。
- 使用 Lip Gloss 统一样式（标题、焦点边框、勾选标记、状态条、确认层）。
- 使用 `WindowSizeMsg` 动态计算列宽与高度。
- 目录扫描建议异步（tea.Cmd + goroutine），避免阻塞 UI。

## 错误处理

- 目录不可读：提示错误并留在当前目录。
- 空选择：阻止进入下一步，并提示“未选择任何文件/目录”。
- 扫描失败：确认层显示失败原因并允许取消。

## 测试策略

- UI 单元测试：选择/取消、焦点切换、`N` 快捷进入、右栏删除。
- 逻辑测试：目录展开与输出路径规则（目录平铺/单文件平铺）。
- 回归测试：原有 Pipeline 与编码逻辑不变。

## 风险与缓解

- 大目录扫描耗时：异步扫描 + 状态提示。
- 用户误解输出规则：确认层清晰列出“输出目录列表”。
